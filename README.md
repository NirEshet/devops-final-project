## **# DevOps Final Project - NameGen Application on AWS EKS**## ## **## Project Overview**## ## **This project demonstrates a complete DevOps deployment workflow for a containerized Node.js application running on AWS EKS.**## ## **The application generates random names and stores data in MongoDB. The entire solution is deployed on Kubernetes, exposed through an AWS Network Load Balancer (NLB), and automatically deployed using GitHub Actions.**## ## **---**## ## **## Architecture**## ## **The deployment flow is:**## ## **GitHub Repository → GitHub Actions → AWS ECR → AWS EKS → NameGen Application → MongoDB StatefulSet**## ## **The project includes:**## ## **\* AWS EKS Cluster (Auto Mode)**## **\* AWS ECR Repository**## **\* Kubernetes Deployments and Services**## **\* MongoDB StatefulSet**## **\* Persistent Volume Storage**## **\* Network Load Balancer (NLB)**## **\* GitHub Actions CI/CD Pipeline**## ## **---**## ## **## Technologies Used**## ## **\* AWS EKS**## **\* AWS ECR**## **\* Kubernetes**## **\* Docker**## **\* GitHub Actions**## **\* MongoDB**## **\* Node.js**## **\* eksctl**## ## **---**## ## **## Repository Structure**## ## **```text**## **.**## **├── .github/workflows/**## **│ NameGen application is deployed as a Kubernetes Deployment with two replicas for high availability.**## ## **The application is exposed externally using a Kubernetes Service of type LoadBalancer, which automatically provisions an AWS Network Load Balancer.**## ## **### Deployment**## ## **```bash**## **kubectl apply -f k8s/app-deployment.yaml**## **kubectl apply -f k8s/app-service.yaml**## **```**## ## **---**## ## **## MongoDB Deployment**## ## **MongoDB is deployed using a Kubernetes StatefulSet.**## ## **The StatefulSet provides:**## ## **\* Stable pod identity**## **\* Persistent storage**## **\* Reliable database deployment**## ## **### Deployment**## ## **```bash**## **kubectl apply -f k8s/mongodb-service.yaml**## **kubectl apply -f k8s/mongodb-statefulset.yaml**## **```**## ## **---**## ## **## Persistent Storage**## ## **Persistent storage is implemented using:**## ## **\* Persistent Volume (PV)**## **\* Persistent Volume Claim (PVC)**## **\* StorageClass (gp3)**## ## **This ensures database data is retained even if pods are recreated.**## ## **---**## ## **## CI/CD Pipeline**## ## **A GitHub Actions workflow automatically deploys changes whenever code is pushed to the main branch.**## ## **The workflow performs the following steps:**## ## **1. Checkout source code**## **2. Configure AWS credentials**## **3. Authenticate to Amazon ECR**## **4. Build Docker image**## **5. Push image to ECR**## **6. Update EKS configuration**## **7. Restart Kubernetes deployment**## ## **Workflow file:**## ## **```text**## **.github/workflows/deploy.yml**## **```**## ## **---**## ## **## Validation**## ## **Useful commands used during deployment and validation:**## ## **```bash**## **kubectl get pods**## **kubectl get services**## **kubectl get deployments**## **kubectl get statefulsets**## **kubectl get pv**## **kubectl get pvc**## **```**## ## **---**## ## **## Screenshots**## ## **The repository contains screenshots demonstrating:**## ## **\* GitHub Repository**## **\* Successful GitHub Actions workflow**## **\* AWS EKS Cluster**## **\* AWS ECR Repository**## **\* Kubernetes Pods**## **\* Kubernetes Deployments**## **\* Kubernetes Services**## **\* MongoDB StatefulSet**## **\* Persistent Volume**## **\* Running Application through the Load Balancer**## ## **---**## ## **Nir Eshet**## ## **DevOps Final Project**
