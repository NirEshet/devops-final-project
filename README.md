## **# DevOps Final Project - NameGen Application**## ## **## Overview**## ## **This project demonstrates a complete DevOps deployment workflow for a containerized Node.js application running on AWS EKS.**## ## **The application generates random names and stores data in a MongoDB database. The project includes Kubernetes manifests, persistent storage, a Network Load Balancer (NLB), and a GitHub Actions CI/CD pipeline.**## ## **## Architecture**## ## **GitHub Repository → GitHub Actions → AWS ECR → AWS EKS → NameGen Application → MongoDB StatefulSet**## ## **The architecture diagram can be found in the `diagrams` folder.**## ## **## Technologies Used**## ## **\* AWS EKS (Auto Mode)**## **\* AWS ECR**## **\* Kubernetes**## **\* Docker**## **\* GitHub Actions**## **\* MongoDB**## **\* Node.js**## **\* eksctl**## ## **## Repository Structure**## ## **```text**## **.**## **├── .github/workflows**## **├── diagrams**## **├── eks**## **├── k8s**## **├── screenshots**## **├── namegen**## **└── README.md**## **```**## ## **## Application Components**## ## **### NameGen Application**## ## **\* Node.js application**## **\* Runs as a Kubernetes Deployment**## **\* Exposed through a Kubernetes LoadBalancer Service**## **\* Uses AWS Network Load Balancer (NLB)**## ## **### MongoDB**## ## **\* Deployed as a StatefulSet**## **\* Uses Persistent Volume storage**## **\* Accessible internally through a Kubernetes Service**## ## **## Kubernetes Resources**## ## **The project deploys the following resources:**## ## **\* Deployment**## **\* Service**## **\* StatefulSet**## **\* Persistent Volume**## **\* Persistent Volume Claim**## **\* StorageClass**## ## **All Kubernetes manifests are located in the `k8s` directory.**## ## **## CI/CD Pipeline**## ## **GitHub Actions automatically performs deployment whenever code is pushed to the `main` branch.**## ## **Pipeline steps:**## ## **1. Checkout source code**## **2. Configure AWS credentials**## **3. Login to Amazon ECR**## **4. Build Docker image**## **5. Push image to ECR**## **6. Connect to EKS cluster**## **7. Restart Kubernetes deployment**## ## **Workflow file:**## ## **```text**## **.github/workflows/deploy.yml**## **```**## ## **## Deployment Verification**## ## **The following screenshots are included in the repository:**## ## **\* GitHub Repository**## **\* Successful GitHub Actions run**## **\* AWS EKS Cluster**## **\* AWS ECR Repository**## **\* Kubernetes Pods**## **\* Kubernetes Deployments**## **\* Kubernetes Services**## **\* MongoDB StatefulSet**## **\* Persistent Volume**## **\* Running Application through Load Balancer**## ## **All screenshots can be found in the `screenshots` folder.**## ## **## Useful Commands**## ## **```bash**## **kubectl get pods**## **kubectl get deployments**## **kubectl get services**## **kubectl get statefulset**## **kubectl get pv**## **kubectl get pvc**## **```**## ## **Nir Eshet**## ## **DevOps Final Project**
